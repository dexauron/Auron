# Auron Finance — память проекта

## Что это
PWA для финансового учёта розничного магазина. Владелец — не программист.
Общение с ним — на русском, простым языком, без жаргона.

## ⚠️ Главное правило
Проект проходит редизайн «с нуля» по этапам. **Не добавляй фичи и не пиши код,
пока соответствующий документ в `docs/` не утверждён владельцем.**

## Память проекта — в документах
Вся история решений живёт в `docs/`, а не в чатах. В начале сессии прочитай:

| Файл | Содержание | Статус |
|------|-----------|--------|
| `docs/00-VISION.md` | Что делает продукт, для кого, чего НЕ делает | ✅ утверждён |
| `docs/01-USER.md` | Портрет пользователя, его день | ✅ утверждён |
| `docs/02-FLOWS.md` | Ключевые сценарии по шагам | 🔄 обсуждается |
| `docs/03-SCREENS.md` | Описание каждого экрана словами | ⬜ не начат |
| `docs/04-DESIGN-SYSTEM.md` | Цвета, кнопки, отступы — правила | ⬜ не начат |
| `docs/05-ARCHITECTURE.md` | Данные, синхронизация, офлайн | ⬜ не начат |
| `docs/DECISIONS.md` | Журнал решений с причинами | 🔄 ведётся |

**Текущий этап: Разговор 2 — Сценарии (`02-FLOWS.md`).**

## Как работать
1. Прочитай документы → пойми, на каком этапе проект
2. Обсуждай с владельцем ТОЛЬКО текущий этап, не прыгай вперёд
3. Каждое принятое решение — сразу в соответствующий документ + строка в `DECISIONS.md`
4. Обновляй статусы и «Текущий этап» в этой таблице
5. Коммить документы после каждого обсуждения (ветка `claude/optimistic-einstein-Afzmv`, мерж в `main` — только по просьбе владельца)
6. Документ больше 2 страниц = решения недостаточно чёткие, сжимай

## Технический контекст (текущее состояние кода)
- `app/` — SPA в одном файле `index.html` (~7000 строк), деплой на GitHub Pages через `.github/workflows/deploy.yml` → https://dexauron.github.io/Auron/
- Backend: Supabase (PostgreSQL + Auth + PostgREST), API-слой в `app/js/api.js`, auth в `app/js/auth.js`
- PWA: `app/sw.js`, версию кэша (`auron-vNN`) бампать при каждом изменении фронта
- Существующий код НЕ трогаем до завершения этапов проектирования
