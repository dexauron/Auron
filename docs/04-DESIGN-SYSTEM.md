# 04 — Дизайн-система

> Статус: 🔄 в работе. Основа утверждена.
> Эталоны: Monzo · Revolut · Zenmoney. Принцип: минимализм без урезания функционала.

---

## Философия

**Прятать сложность, не убирать её.** Главное видно сразу — детали раскрываются по тапу. Пользователь не должен думать где что лежит. Каждый элемент на экране либо несёт информацию, либо даёт действие — лишнего нет.

## Цвета

### Светлая тема (по умолчанию)
```
--bg-primary:     #F2F2F7   /* фон страницы */
--bg-card:        #FFFFFF   /* карточки, листы */
--bg-secondary:   #F2F2F7   /* вложенные блоки внутри карточки */
--text-primary:   #1C1C1E   /* заголовки, суммы */
--text-secondary: #8E8E93   /* подписи, даты, метки */
--text-tertiary:  #C7C7CC   /* плейсхолдеры, неактивное */
--separator:      #E5E5EA
```

### Тёмная тема (авто по системным настройкам)
```
--bg-primary:     #000000
--bg-card:        #1C1C1E
--bg-secondary:   #2C2C2E
--text-primary:   #FFFFFF
--text-secondary: #8E8E93
--text-tertiary:  #48484A
--separator:      #38383A
```

### Акценты (одинаковы в обеих темах)
```
--brand:    #5E5CE6   /* индиго — кнопки, ссылки, активные вкладки */
--income:   #30D158   /* зелёный — доходы, успех */
--expense:  #FF453A   /* красный — расходы, ошибки, долги */
--transfer: #FF9F0A   /* оранжевый — переводы, предупреждения */
--neutral:  #8E8E93   /* серый — нейтральное, архив */
```

### Правила применения
- Доход → `--income`, расход → `--expense`, перевод → `--transfer`
- Расхождение кассы → `--expense` (отрицательное) / `--income` (нулевое)
- Накопления: в норме → `--income`, отстают → `--transfer`, критично → `--expense`
- `--brand` → только интерактивные элементы (кнопки, ссылки, активные состояния)

## Отступы и сетка

Базовая единица: **8px**. Все отступы кратны 8.

```
4px    между иконкой и текстом
8px    между элементами внутри карточки
16px   отступ контента от краёв / поля экрана
24px   между карточками
32px   между секциями
```

Скругления: **16px** (карточки) · **12px** (элементы) · **8px** (теги).

## Компоненты

### Кнопки
```
Первичная   фон --brand, текст белый, высота 50px, радиус 14px, 17px/600
Вторичная   фон --bg-secondary, текст --brand, те же размеры
Опасная     фон --expense 10%, текст --expense, те же размеры
Текстовая   без фона, текст --brand, высота 44px
```
В нижних листах — полная ширина. Несколько кнопок — стопкой, зазор 8px.

### Карточки
```
фон --bg-card · скругление 16px · отступ 16px · тени нет
```
Разделение карточек — цветом фона страницы (серый), не тенями.

### Нижний лист (Bottom Sheet)
```
Рукоятка: 36×4px, радиус 2px, цвет --separator, сверху 8px по центру
Фон: --bg-card · скругление верха 24px · макс. высота 90%
Фон за листом: чёрный 40% · закрытие: свайп вниз или тап на фон
```

### FAB (кнопка [+])
```
56×56px · радиус 16px · цвет --brand · иконка + белая 24px
правый нижний угол, 16px от краёв, 16px над навигацией
тень: 0 4px 16px rgba(94, 92, 230, 0.4)
```

### Строка транзакции
```
[иконка 40×40px]  [название · комментарий]  [±сумма]
                  [дата · счёт           ]
высота: 64px · разделитель — только отступ, без линии
```
Группировка: «Сегодня» / «Вчера» / «12 июня» — серый заголовок 13px + дневные итоги.

### Иконки категорий
Круг 40×40px · цветной фон · белая эмодзи по центру.
```
Продукты 🛒 #0EA5E9 · Кафе ☕ #F59E0B · Транспорт 🚗 #8B5CF6
Зарплата 💳 #10B981 · Аренда 🏠 #F59E0B · Закупка 📦 #0EA5E9
Коммуналка 💡 #EAB308 · Налоги 🏛 #DC2626 · Перевод ↔ #FF9F0A · Прочее 📋 #64748B
```

### Поля ввода
```
фон --bg-secondary · радиус 12px · отступ 16px · высота 50px
лейбл: 13px --text-secondary над полем
активное: рамка 2px --brand · ошибка: рамка 2px --expense + текст ошибки снизу
```

### Нумпад
```
4 колонки × 4 строки · высота кнопки 64px · цифры 24px/500
сумма над нумпадом: hero-размер по центру, обновляется мгновенно
```

### Навигационная панель
```
высота 83px · фон --bg-card + blur(20px)
активная вкладка: иконка + текст --brand + точка 3px снизу
неактивная: --text-tertiary
бейдж: 8px круг --expense, правый верхний угол иконки
```

### Toast
```
снизу над навигацией, 16px отступ · фон #1C1C1E · текст белый 15px · радиус 14px
появление: slide up + fade 200ms · исчезает через 2.5 сек
успех: ✓ · ошибка: ⚠️ + красная полоска слева 4px
```

### Прогресс-бар
```
трек --bg-secondary · высота 6px · радиус 3px
≥70% → --income · ≥40% → --transfer · <40% → --expense
```

### Скелетон
```
цвет --bg-secondary · shimmer-анимация 1.5s
только при первой загрузке — обновление через оптимистичный UI
```

## UX-паттерны

| Паттерн | Правило |
|---------|---------|
| Bottom Sheet | Все действия на телефоне — снизу. Не новые экраны. |
| FAB | Всегда видна на Главной и Операциях. |
| Оптимистичный UI | Обновить локально мгновенно → сервер → откат + Toast при ошибке |
| Свайп влево | На строке → Редактировать / Удалить |
| Свайп в нумпаде | Влево/вправо = Расход → Доход → Перевод |
| Долгое нажатие | На транзакции → «Повторить» |
| Тап на цифру | Прямое редактирование любого числа на экране — без кнопки «Редактировать» |
| Pull-to-refresh | Потянуть вниз → обновить |
| Pull-to-search | Потянуть вниз на любом экране → глобальный поиск |
| Пустые состояния | Иконка + 1 строка текста + кнопка первого действия |
| Enter / Esc | Сохранить / Закрыть (веб и планшет) |
| Группировка | Сегодня / Вчера / дата + дневные итоги |
| Период-пиллы | Сегодня · Неделя · Месяц · Год · Свой |
| Умная клавиатура | На поле суммы — цифровая клавиатура автоматически, без переключения |
| Черновик | Незавершённая форма сохраняется, красная точка на FAB — «есть несохранённое» |
| Shake to Undo | Потряс телефон → «Отменить последнее действие?» |
| Калькулятор | В нумпаде работают +/- : ввести «15000+8500» → считает сумму |
| Живой учебник | Каждая функция объясняется один раз при первом открытии — больше не повторяется |

## Адаптивное поведение

### Тихий старт
Первые 14 дней приложение только наблюдает — никаких советов и предупреждений кроме критических. Пользователь работает в тишине, система учится его паттернам.

```
Дни  1–14   Тишина. Только критические ошибки (уход в минус и т.д.)
Дни 15–30   Лёгкие подсказки. Максимум 1–2 в день.
Месяц 2+    Полноценные инсайты, аномалии, рекомендации.
```

### Правило трёх
Пользователь трижды закрыл одно и то же предупреждение → приложение предлагает его отключить. После отклонения — никогда не возвращается к нему само.

### Прогрессивное раскрытие
Три уровня сложности — переключаются в настройках, ИИ предлагает переход сам:

| Уровень | Когда | Что видно |
|---------|-------|-----------|
| Простой | Дни 1–14 (авто) | Только ключевые функции |
| Стандартный | С 3-й недели (авто) | Полный интерфейс |
| Расширенный | По выбору пользователя | Всё + горячие клавиши + редкие функции |

### Адаптивная Главная
Карточки и разделы переставляются по частоте использования. Функция не используемая 30+ дней — уходит вниз (не удаляется). ИИ раз в месяц предлагает скрыть неиспользуемые разделы и вынести вперёд частые.

### Полный контроль пользователя
Каждое уведомление, каждое предупреждение, каждый ИИ-инсайт — отдельно включается/выключается в настройках. Порог срабатывания предупреждений настраивается (например: «необычная сумма» — от какого порога считать необычной).

### Контекстная осведомлённость
Приложение знает время суток и подстраивается:
- 7:00–11:00 → утренний режим: первым делом предлагает Z-отчёт
- 19:00–23:00 → вечерний режим: предлагает итог дня и отправить отчёт

Контекстная подсказка «Обычно сейчас вы делаете X → [Открыть]» появляется только после **20+ повторений** одного действия в одно время. Раньше — тишина.

## Доверие и прозрачность

- **«Откуда эта цифра?»** — тап на любое число → формула как оно считается
- **Источник данных** — под прогнозом: «на основе 47 записей, точность 78%»
- **Уровень уверенности** — ИИ показывает насколько уверен в прогнозе визуально
- **Аудит везде** — любое изменение любой записи фиксируется: кто, когда, было→стало
- Технические коды ошибок пользователю не показываются — только человеческое объяснение + конкретный следующий шаг

---

## Умные уведомления

- **Пакетирование** — несколько событий = одно уведомление утром, не 5 отдельных
- **Обучение** — система замечает в какое время пользователь читает уведомления, перестаёт слать в игнорируемое время
- **Действие из уведомления** — кнопки прямо в уведомлении: `[Оплачено]` `[Перенести]` без открытия приложения
- **Умное время** — не отправлять уведомление если пользователь сейчас активен в приложении
- Максимум уведомлений в день: **3**. Всё остальное — в центр уведомлений внутри приложения

---

## Доступность

- **Крупный текст** — интерфейс перестраивается корректно при системном увеличении шрифта, не ломается
- **Дальтонизм** — пульс 🟢🟡🔴 дублируется формами ●▲✕; графики — паттернами, не только цветом
- **Упрощённый режим** — специально для нетехнических пользователей: 3 кнопки на главной, только самые нужные цифры, никаких настроек
- **Минимум анимаций** — опция для чувствительных к движению (prefers-reduced-motion)

---

## Многоустройственность

- **Планшет** — два столбца: список слева, детали справа. Бухгалтер видит всё без переходов
- **Передача между устройствами** — черновик формы синхронизируется мгновенно; начал на телефоне, продолжил на планшете
- **Печать** — любой отчёт и карточка поставщика → кнопка `[🖨 Печать]` → чистая вёрстка без UI-элементов
- **PWA-виджет** — пульс + выручка + касса прямо на рабочем столе без открытия приложения
- **Шорткаты на иконке** — долгое нажатие → Z-отчёт / Записать выплату / Новая транзакция

## Надёжность и защита от ошибок

### Предотвращение
- Валидация в реальном времени — не после нажатия «Сохранить», а при вводе
- Человеческие сообщения об ошибках: «Сумма больше остатка на ₽12 000» — не коды
- Предупреждение перед опасным действием с конкретикой: сколько записей затронет
- Защита от случайных нулей: сумма в 10× от обычной → мягкий вопрос

### Сохранность данных
- Автосохранение черновиков каждые 30 секунд
- Корзина с восстановлением в течение 30 дней
- История версий каждой записи с диффом изменений
- Предложение экспорта перед архивированием или удалением

### Офлайн и синхронизация
- Полноценная работа без интернета — данные копятся в очереди
- Индикатор статуса: 🟢 синхронизировано · 🟡 синхронизируется · 🔴 нет связи (N записей в очереди)
- При конфликте двух пользователей — показывает обе версии, человек выбирает

### Целостность
- Проверка балансов в фоне раз в день — при расхождении предупреждение
- Чеклист закрытия месяца: незакрытые дни табеля, Z-отчёты без кассира, неоплаченные запланированные выплаты
- Умные подсказки вместо технических ошибок + кнопка «Попробовать снова»
