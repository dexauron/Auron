# 03 — Экраны

> Статус: 🔄 в работе.
> Каждый экран: зачем существует, что на нём сверху вниз, какие действия доступны.
> Если экран не обслуживает ни один сценарий из 02-FLOWS.md — его не существует.

---

## Навигация

Приложение работает на телефоне (администратор, владелец) и на компьютере/планшете (бухгалтер).
На телефоне — нижняя панель навигации. На компьютере — боковая панель слева.

### Основные разделы:

| # | Раздел | Для кого активен |
|---|--------|-----------------|
| 1 | **Главная** | Все |
| 2 | **Касса** | Бухгалтер |
| 3 | **Поставщики** | Бухгалтер + Администратор |
| 4 | **Персонал** | Бухгалтер |
| 5 | **Финансы** | Бухгалтер + Владелец |
| 6 | **Товары** | Все (только если включён ручной режим учёта) |

**Настройки** — иконка шестерёнки в углу, не основной раздел.

### Что видит каждая роль:
- **Бухгалтер:** все разделы + Настройки
- **Администратор:** Главная + Поставщики (только заказы и запись на выплату)
- **Владелец:** Главная (расширенная) + Финансы (только чтение)

---

## Экран 1: Главная

**Зачем:** сводка дня — первый экран при открытии. Бухгалтер видит всё важное за секунду, все действия раскрываются прямо здесь без смены экрана.

**Состав сверху вниз:**

```
Название магазина                    [⚙️]

Наличных в кассе
₽ 48 500                    Ср, 11 июн
Выручка сегодня      Свободно
₽ 62 000             ₽ 15 200
```
> «Свободно» = наличка − запланированные выплаты − сумма к пополнению накоплений

---

**Карточка Z-отчёта** (только если смена сегодня не заполнена):
```
┌──────────────────────────────────┐
│  🧾 Смена не закрыта             │
│  Внести Z-отчёт             [→] │
└──────────────────────────────────┘
```
Исчезает после заполнения. Исправить смену можно в любое время в разделе «Касса» — каждое изменение фиксируется в журнале аудита.

---

**Карточка «Выплаты сегодня»** (свёрнута по умолчанию):
```
┌──────────────────────────────────┐
│  Выплаты сегодня                 │
│  4 поставщика  •  ₽ 94 000  [↓] │
└──────────────────────────────────┘
```
Тап [↓] → нижний лист со списком (экран не меняется):
```
┌──────────────────────────────────┐
│  Выплаты сегодня           [✕]  │
│  Иванов П.         ₽ 35 000  >  │
│  ООО Альфа         ₽ 28 000  >  │
│  Петров К.         ₽ 18 000  >  │
│  Нурмат ИП         ₽ 13 000  >  │
└──────────────────────────────────┘
```
Тап на поставщике → действия (в том же листе):
```
┌──────────────────────────────────┐
│  ← Иванов П.       Долг: ₽ 35К  │
│  [💸 Оплатить]                   │
│  [📅 Перенести]                  │
│  [✗ Не пришёл]                  │
│  [🗑 Отменить запись]            │
└──────────────────────────────────┘
```

---

**Карточка «Ожидаемые заказы»** (только если есть заказы на сегодня):
```
┌──────────────────────────────────┐
│  Ожидаемые заказы                │
│  Молочка • Иванов П.   ~₽ 15К > │
│  Бакалея • ООО Альфа   ~₽ 22К > │
└──────────────────────────────────┘
```

---

**Карточка «Накопления»:**
```
┌──────────────────────────────────┐
│  Накопления                      │
│  Аренда    ████████░░  ₽42К/₽60К │
│  Налоги    █████░░░░░  ₽18К/₽45К │
│  Зарплата  ██████████  ₽90К/₽90К │
└──────────────────────────────────┘
```

---

**Пустое состояние** (первый запуск): карточки выплат и накоплений не показываются. Только карточка Z-отчёта и приглашение начать работу.

**Принцип:** все действия на главной раскрываются в нижние листы — основной экран не меняется.

## Экран 1а: Сводка по всем магазинам

**Зачем:** владелец и главный бухгалтер при входе видят всю сеть сразу — без выбора магазина.  
**Кто видит:** только аккаунты с доступом к нескольким магазинам. Обычный сотрудник этот экран не видит — попадает сразу в свой магазин.

**Состав:**

```
Все магазины              Ср, 11 июн   [+ Магазин]

Общая выручка сегодня     Общий долг поставщикам
₽ 186 000                 ₽ 420 000

─── Магазины ───
┌──────────────────────────────────┐
│  Way Market №1                 > │
│  Выручка ₽112К  •  Касса ₽48К   │
│  ⚠️ Смена не закрыта            │
└──────────────────────────────────┘
┌──────────────────────────────────┐
│  Way Market №2                 > │
│  Выручка ₽74К   •  Касса ₽31К   │
│  ✓ Смена закрыта                │
└──────────────────────────────────┘
```

Тап на магазине → переход внутрь, все экраны этого магазина (Главная, Касса, Поставщики...).  
Вернуться к сводке — кнопка «← Все магазины» в шапке.

**Карточка магазина показывает:**
- Название
- Выручка сегодня + наличка в кассе
- Статус смены (закрыта / не закрыта — чтобы не пропустить)

## Экран 2: Касса

**Зачем:** ввод Z-отчёта по смене и история всех смен. Сценарий 1.  
**Кто:** только Бухгалтер.

**Вкладки внутри раздела:** `[Новая смена]` `[История]`

---

### Вкладка «Новая смена»

```
Дата      [11.06.2025]
Смена     [Дневная ▼]       Кассир  [Иванова А. ▼]

─── По Z-отчёту ───
Наличные          ₽ [________]
Карта             ₽ [________]
Переводы (СБП)    ₽ [________]

─── Фактически в кассе ───
Наличные          ₽ [________]

─── Выплаты с кассы ───
[+ Добавить выплату]
  Инкассация  ₽ 20 000  [✕]
  Иман        ₽  3 500  [✕]

─── Итог ───
Расхождение: 🔴 -₽ 300
(Z-отчёт − (Факт + Выплаты + Иман))

[Сохранить смену]
```

После сохранения: карточка Z-отчёта на Главной исчезает.

---

### Вкладка «История»

Последние 7 дней — видно сразу. Дальше — свёрнуто по месяцам.

```
─── Сегодня, 11 июн ───
Дневная  Иванова А.  ₽62К  ✓ 0₽     >
Ночная   Петров В.   ₽31К  🔴 -₽300  >

─── Вчера, 10 июн ───
Дневная  Иванова А.  ₽58К  ✓ 0₽     >
...

─── Май 2025 ──────────────────── [↓] ───
─── Апрель 2025 ───────────────── [↓] ───
```

Тап на смене → детали: все введённые суммы + история исправлений (кто, когда, что изменил).  
Кнопка «Редактировать» — открывает форму с заполненными данными. Изменение фиксируется в аудите.

**Пустое состояние:** «Смены ещё не вводились. Начни с кнопки Z-отчёта на главной.»

## Экран 3: Поставщики

**Зачем:** управление долгами, выплатами и заказами. Сценарии 2, 3.  
**Кто:** Бухгалтер (полный доступ), Администратор (только заказы + запись на выплату).

**Вкладки:** `[Выплаты]` `[Поставщики]` `[Заказы]`  
По умолчанию открывается **Выплаты**.

---

### Вкладка «Выплаты»

```
─── Сегодня ───────────────────────
Иванов П.        ₽ 35 000   Долг: ₽120К  >
ООО Альфа        ₽ 28 000   Долг: ₽ 55К  >
Петров К.        ₽ 18 000   Долг: ₽ 90К  >

─── Завтра ────────────────────────
Нурмат ИП        ₽ 13 000   Долг: ₽ 42К  >
Салим ИП         ₽  8 000   Долг: ₽ 28К  >

─── Позже ─────────────────────────
15 июн  Рустам ИП  ₽ 22 000  >
18 июн  ООО Бета   ₽ 45 000  >
```

Тап на записи → нижний лист с действиями:
```
┌──────────────────────────────────┐
│  ← Иванов П.       Долг: ₽120К  │
│  Запланировано: ₽35 000, сегодня │
│                                  │
│  [💸 Оплатить]                   │
│  [📅 Перенести]                  │
│  [✗ Не пришёл]                  │
│  [🗑 Отменить запись]            │
└──────────────────────────────────┘
```

«Оплатить» → форма: сумма оплаты / погашение долга / новый долг (накладная). Долг пересчитывается автоматически.

«Перенести» → варианты: на завтра / на завтра с увеличением суммы / на конкретную дату / освободить деньги.

[+ Записать выплату] — кнопка добавить новую запись (поставщик + сумма + дата).

## Экран 4: Персонал

**Зачем:** управление сотрудниками, табель, расчёт зарплаты. Сценарий 5.  
**Кто:** только Бухгалтер.

**Вкладки:** `[Табель]` `[Сотрудники]` `[Зарплата]`

---

### Вкладка «Табель»

Открывается на сегодняшнем дне. Статусы авто-заполняются по графику каждого сотрудника — бухгалтер проверяет и подтверждает.

```
◀  11 июня, среда  ▶        [Подтвердить день ✓]

Сафина Г.    [Вышла ▼]            5/2, рабочий
Петров В.    [Выходной ▼]         2/2, выходной
Иванова А.   [Отпуск ▼]           событие до 28 июн
Нурмат К.    [Вышел ▼]   [× __]  2/2, рабочий (коэф.)
Рустам М.    [Полдня ▼]           вручную изменено
```

Стрелки ◀ ▶ — перейти на другой день.  
Тап на статусе → выбор из списка: **Вышел / Полдня / Выходной / Больничный / Прогул / Отпуск**  
Поле коэффициента `[× __]` — скрыто по умолчанию, появляется при нестандартной частичной оплате (0.75, 0.3 и т.д.)  
«Подтвердить день» — закрывает день, данные учитываются в расчёте зарплаты.

Незакрытые дни отображаются с предупреждением при расчёте зарплаты в конце месяца.

---

### Вкладка «Сотрудники»

```
[+ Добавить сотрудника]

Сафина Г.      Продавец    ₽1 500/день   активна  >
Петров В.      Кладовщик   ₽180/час      активен  >
Иванова А.     Продавец    ₽1 500/день   отпуск   >
─────────────────────────────────────────────────
Уволенные  [↓]
  Алиев Р.   Продавец   уволен 03.05.2025          >
```

Тап на сотруднике → карточка сотрудника:
```
Сафина Гульнара
Продавец  •  ₽1 500/день

График:  [5/2 ▼]
─── Временные события ──────────────
  Нет активных событий
  [+ Добавить]  (отпуск / больничный / с даты по дату)

─── Авансы ─────────────────────────
  05.06  Аванс  ₽10 000
  [+ Выдать аванс]

─── История выплат ──────────────────
  31.05  Зарплата  ₽32 000
  28.02  Зарплата  ₽31 500

[✏️ Редактировать]  [🚫 Уволить]
```

«Уволить» → указать дату увольнения → сотрудник уходит в раздел «Уволенные», история сохраняется.

---

### Вкладка «Зарплата»

Открывается в конце месяца. Показывает расчёт по каждому сотруднику.

```
Расчёт за май 2025

Сафина Г.    22 дн × ₽1 500  = ₽33 000
             Аванс              -₽10 000
             Итого:              ₽23 000  [Выплатить]

Петров В.    168 ч × ₽180    = ₽30 240
             Аванс              -₽5 000
             Итого:              ₽25 240  [Выплатить]

─── Незакрытые дни ─────────────────
⚠️ 3 дня не подтверждены → уточни табель
```

Кнопка «Выплатить» → сумма редактируемая (можно скорректировать вручную) → подтвердить → записывается в историю выплат.

**Пустое состояние:** «Добавь первого сотрудника чтобы начать вести табель.»
