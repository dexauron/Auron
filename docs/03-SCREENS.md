# 03 — Экраны

> Статус: 🔄 в работе.
> Каждый экран: зачем существует, что на нём сверху вниз, какие действия доступны.
> Если экран не обслуживает ни один сценарий из 02-FLOWS.md — его не существует.

---

## Навигация

Приложение работает на телефоне (администратор, владелец) и на компьютере/планшете (бухгалтер).
На телефоне — нижняя панель навигации. На компьютере — боковая панель слева.

### Основные разделы:

| # | Раздел | Для кого активен |
|---|--------|-----------------|
| 1 | **Главная** | Все |
| 2 | **Касса** | Бухгалтер |
| 3 | **Поставщики** | Бухгалтер + Администратор |
| 4 | **Персонал** | Бухгалтер |
| 5 | **Финансы** | Бухгалтер + Владелец |
| 6 | **Товары** | Все (только если включён ручной режим учёта) |

**Настройки** — иконка шестерёнки в углу, не основной раздел.

### Что видит каждая роль:
- **Бухгалтер:** все разделы + Настройки
- **Администратор:** Главная + Поставщики (только заказы и запись на выплату)
- **Владелец:** Главная (расширенная) + Финансы (только чтение)

---

## Экран 1: Главная

**Зачем:** сводка дня — первый экран при открытии. Бухгалтер видит всё важное за секунду, все действия раскрываются прямо здесь без смены экрана.

**Состав сверху вниз:**

```
Название магазина                    [⚙️]

Наличных в кассе
₽ 48 500                    Ср, 11 июн
Выручка сегодня      Свободно
₽ 62 000             ₽ 15 200
```
> «Свободно» = наличка − запланированные выплаты − сумма к пополнению накоплений

---

**Карточка Z-отчёта** (только если смена сегодня не заполнена):
```
┌──────────────────────────────────┐
│  🧾 Смена не закрыта             │
│  Внести Z-отчёт             [→] │
└──────────────────────────────────┘
```
Исчезает после заполнения. Исправить смену можно в любое время в разделе «Касса» — каждое изменение фиксируется в журнале аудита.

---

**Карточка «Выплаты сегодня»** (свёрнута по умолчанию):
```
┌──────────────────────────────────┐
│  Выплаты сегодня                 │
│  4 поставщика  •  ₽ 94 000  [↓] │
└──────────────────────────────────┘
```
Тап [↓] → нижний лист со списком (экран не меняется):
```
┌──────────────────────────────────┐
│  Выплаты сегодня           [✕]  │
│  Иванов П.         ₽ 35 000  >  │
│  ООО Альфа         ₽ 28 000  >  │
│  Петров К.         ₽ 18 000  >  │
│  Нурмат ИП         ₽ 13 000  >  │
└──────────────────────────────────┘
```
Тап на поставщике → действия (в том же листе):
```
┌──────────────────────────────────┐
│  ← Иванов П.       Долг: ₽ 35К  │
│  [💸 Оплатить]                   │
│  [📅 Перенести]                  │
│  [✗ Не пришёл]                  │
│  [🗑 Отменить запись]            │
└──────────────────────────────────┘
```

---

**Карточка «Ожидаемые заказы»** (только если есть заказы на сегодня):
```
┌──────────────────────────────────┐
│  Ожидаемые заказы                │
│  Молочка • Иванов П.   ~₽ 15К > │
│  Бакалея • ООО Альфа   ~₽ 22К > │
└──────────────────────────────────┘
```

---

**Карточка «Накопления»:**
```
┌──────────────────────────────────┐
│  Накопления                      │
│  Аренда    ████████░░  ₽42К/₽60К │
│  Налоги    █████░░░░░  ₽18К/₽45К │
│  Зарплата  ██████████  ₽90К/₽90К │
└──────────────────────────────────┘
```

---

**Пустое состояние** (первый запуск): карточки выплат и накоплений не показываются. Только карточка Z-отчёта и приглашение начать работу.

**Принцип:** все действия на главной раскрываются в нижние листы — основной экран не меняется.

## Экран 1а: Сводка по всем магазинам

**Зачем:** владелец и главный бухгалтер при входе видят всю сеть сразу — без выбора магазина.  
**Кто видит:** только аккаунты с доступом к нескольким магазинам. Обычный сотрудник этот экран не видит — попадает сразу в свой магазин.

**Состав:**

```
Все магазины              Ср, 11 июн   [+ Магазин]

Общая выручка сегодня     Общий долг поставщикам
₽ 186 000                 ₽ 420 000

─── Магазины ───
┌──────────────────────────────────┐
│  Way Market №1                 > │
│  Выручка ₽112К  •  Касса ₽48К   │
│  ⚠️ Смена не закрыта            │
└──────────────────────────────────┘
┌──────────────────────────────────┐
│  Way Market №2                 > │
│  Выручка ₽74К   •  Касса ₽31К   │
│  ✓ Смена закрыта                │
└──────────────────────────────────┘
```

Тап на магазине → переход внутрь, все экраны этого магазина (Главная, Касса, Поставщики...).  
Вернуться к сводке — кнопка «← Все магазины» в шапке.

**Карточка магазина показывает:**
- Название
- Выручка сегодня + наличка в кассе
- Статус смены (закрыта / не закрыта — чтобы не пропустить)

## Экран 2: Касса

**Зачем:** ввод Z-отчёта по смене и история всех смен. Сценарий 1.  
**Кто:** только Бухгалтер.

**Вкладки внутри раздела:** `[Новая смена]` `[История]`

---

### Вкладка «Новая смена»

```
Дата      [11.06.2025]
Смена     [Дневная ▼]       Кассир  [Иванова А. ▼]

─── По Z-отчёту ───
Наличные          ₽ [________]
Карта             ₽ [________]
Переводы (СБП)    ₽ [________]

─── Фактически в кассе ───
Наличные          ₽ [________]

─── Выплаты с кассы ───
[+ Добавить выплату]
  Инкассация  ₽ 20 000  [✕]
  Иман        ₽  3 500  [✕]

─── Итог ───
Расхождение: 🔴 -₽ 300
(Z-отчёт − (Факт + Выплаты + Иман))

[Сохранить смену]
```

После сохранения: карточка Z-отчёта на Главной исчезает.

---

### Вкладка «История»

Последние 7 дней — видно сразу. Дальше — свёрнуто по месяцам.

```
─── Сегодня, 11 июн ───
Дневная  Иванова А.  ₽62К  ✓ 0₽     >
Ночная   Петров В.   ₽31К  🔴 -₽300  >

─── Вчера, 10 июн ───
Дневная  Иванова А.  ₽58К  ✓ 0₽     >
...

─── Май 2025 ──────────────────── [↓] ───
─── Апрель 2025 ───────────────── [↓] ───
```

Тап на смене → детали: все введённые суммы + история исправлений (кто, когда, что изменил).  
Кнопка «Редактировать» — открывает форму с заполненными данными. Изменение фиксируется в аудите.

**Пустое состояние:** «Смены ещё не вводились. Начни с кнопки Z-отчёта на главной.»
