# 03 — Экраны

> Статус: 🔄 на финальном согласовании. Все основные экраны описаны, ждут подтверждения владельца.
> Каждый экран: зачем существует, что на нём сверху вниз, какие действия доступны.
> Если экран не обслуживает ни один сценарий из 02-FLOWS.md — его не существует.

## Экран 0а: Регистрация (первый вход)

**Зачем:** новый пользователь создаёт аккаунт за 3 шага — без лишних полей.

```
Шаг 1 — Телефон:
┌─────────────────────────────────┐
│                                 │
│        Auron Finance            │
│                                 │
│  +7 (___) ___-__-__             │
│  ─────────────────────          │
│                                 │
│  [Получить код →]               │
│                                 │
│  Данные хранятся только у вас   │
└─────────────────────────────────┘

Шаг 2 — Код из SMS:
┌─────────────────────────────────┐
│  Код отправлен на +7 900 …      │
│  [Изменить номер]               │  ← кнопка возврата
│                                 │
│        [ _ ][ _ ][ _ ][ _ ]     │
│                                 │
│  Не пришёл? Отправить снова (59)│
└─────────────────────────────────┘

Шаг 3 — PIN и биометрия:
┌─────────────────────────────────┐
│  Создайте PIN для входа         │
│                                 │
│        [ _ ][ _ ][ _ ][ _ ]     │
│                                 │
│  Используется если Face ID      │
│  недоступен                     │
└─────────────────────────────────┘
→ Автоматически предлагает Face ID / Touch ID
→ Готово — переход на Онбординг (новый) или Главную (повторный)
```

**Крайние случаи:**
- Телефон уже зарегистрирован → «Этот номер уже есть в Auron → [Войти]»
- Неверный OTP 3 раза → таймер 30 сек перед новой попыткой
- Неверный OTP 5 раз → запросить новый код (старый аннулируется)
- SMS не пришла → кнопка «Изменить номер» возвращает на шаг 1
- Face ID недоступен на устройстве → шаг пропускается, только PIN

**Правила:**
- Никаких имён, email, паролей при регистрации
- SMS используется **только здесь** — один раз в жизни
- PIN = 4 цифры, не пароль
- После настройки Face ID — SMS больше никогда не нужен при входе

## Экран 0б: Вход (повторный)

**Зачем:** бухгалтер открывает приложение каждое утро — вход должен занимать 1 секунду.

```
Стандартный вход (автоматически):
┌─────────────────────────────────┐
│                                 │
│        Auron Finance            │
│                                 │
│    [Face ID / Touch ID]         │
│    Прикоснитесь к датчику       │
│                                 │
│    [Войти с PIN]                │
└─────────────────────────────────┘
```

**Логика:**
- Открыл приложение → сразу запрос биометрии (без нажатия кнопки)
- Не сработала → PIN 4 цифры
- Face ID выключен в настройках телефона → показываем только PIN, без нерабочей кнопки
- Восстановление забытого PIN → SMS-код (единственный второй случай SMS)
- Смена устройства → SMS-верификация + новый PIN

**Защита от перебора PIN:**
```
1–5 попыток   → без ограничений
6–10 попыток  → таймер 30 сек между попытками
11–15 попыток → таймер 5 минут
16+ попыток   → блокировка, требуется SMS-восстановление
```

## Нижний лист: Быстрый ввод (FAB [+])

**Зачем:** добавить транзакцию за 3 действия — тип, сумма, категория. Сценарий У1.  
**Как открывается:** кнопка [＋] на любом экране (Главная, Операции).

```
┌─────────────────────────────────────┐
│  ▬▬▬  (рукоятка)                    │
│                                     │
│  ● РАСХОД   ○ ДОХОД   ○ ПЕРЕВОД    │
│                                     │
│         ₽ 1 500                     │
│                                     │
│  🛒 Закупка          >              │
│  💵 Наличные         >              │
│  📅 Сегодня, 11 июн  >   [💬]       │
│                                     │
│  [7] [8] [9]  [⌫]                  │
│  [4] [5] [6]                        │
│  [1] [2] [3]  [.]                   │
│     [0]       [✓ Сохранить]         │
└─────────────────────────────────────┘
```

**Принципы:**
- Тип по умолчанию: **Расход** (самый частый)
- Счёт: запоминает последний использованный
- Дата: сегодня — менять не нужно в 95% случаев
- Категория: предлагается автоматически по последней / по ключевым словам в комментарии
- Комментарий `[💬]` — скрыт, появляется по тапу
- Сумма обновляется мгновенно при каждой цифре

**Крайние случаи:**
- Сумма = 0 → кнопка [✓ Сохранить] неактивна, серая
- Нет ни одного счёта → FAB открывает не нумпад, а: «Сначала добавь счёт → [Добавить]»
- Перевод: «Откуда» и «Куда» выбран один счёт → кнопка неактивна + подсказка «Выберите разные счета»
- Калькулятор: ввёл «15000+8500» → нажал = → показывает «₽23 500»

**Перевод между счетами:**
```
  ○ РАСХОД   ○ ДОХОД   ● ПЕРЕВОД

  Откуда: 💵 Наличные    >
  Куда:   💳 Карта        >
  ₽ 20 000
```

**Автокатегория:** написал «продукты» в комментарии → категория подставляется сама.  
**Повторить:** долгое нажатие на транзакцию в истории → «Повторить» → форма заполнена.

## Навигация

Приложение работает на телефоне (администратор, владелец) и на компьютере/планшете (бухгалтер).
На телефоне — нижняя панель навигации. На компьютере — боковая панель слева.

### Принцип: навигация собирается из активных модулей

**Универсальное ядро (есть у всех):**

| # | Раздел | Для кого |
|---|--------|---------|
| 1 | **Главная** | Все |
| 2 | **Операции** | Все — история всех транзакций |
| 3 | **Аналитика** | Все — базовые графики доходов/расходов |
| 4 | **Настройки** | Все |

**Модуль «Розница» заменяет «Аналитика» на «Финансы» и добавляет разделы:**

| # | Раздел | Для кого активен |
|---|--------|-----------------|
| 3* | **Финансы** | Бухгалтер + Владелец (вместо «Аналитика» — расширенная версия с Накопления/Цели/Аналитика/Отчёт) |
| + | **Касса** | Бухгалтер |
| + | **Поставщики** | Бухгалтер + Администратор |
| + | **Персонал** | Бухгалтер |
| + | **Товары** | Все (если включён ручной учёт) |

**Правило: в нижней панели телефона максимум 5 вкладок.** Если модули дают больше — пятая вкладка становится «Ещё» и собирает остальное. Настройки всегда доступны через ⚙️ в шапке — место в панели не занимают.

**Итого для пользователя Way Market (Ритейл):**
Главная · Касса · Поставщики · Финансы · **Ещё**
(в «Ещё»: Операции · Персонал · Товары · Настройки)

**Итого для личного пользователя (только ядро):**
Главная · Операции · Аналитика · Настройки

**Итого для ИП (ядро + модуль Проекты):**
Главная · Операции · Проекты · Аналитика · Настройки

> Новый модуль = новые вкладки в навигации. Модуль Розница заменяет базовую «Аналитику» расширенным разделом «Финансы». Красные точки-бейджи (несделанные задачи) показываются и на вкладке «Ещё», если проблема внутри неё (например, табель не подтверждён).

> На компьютере и планшете ограничения нет — боковая панель показывает все разделы сразу.

---

### Что видит каждая роль (модуль Розница):
- **Бухгалтер:** все разделы + Настройки
- **Администратор:** Главная + Поставщики (только заказы и запись на выплату)
- **Владелец:** Главная (расширенная) + Аналитика (только чтение)

## Экран 1: Главная

**Зачем:** сводка дня — первый экран при открытии. Бухгалтер видит всё важное за секунду, все действия раскрываются прямо здесь без смены экрана.

**Состав сверху вниз:**

---

**Шапка:**
```
🟢  Way Market        [🔔]  [💬 ИИ]  [⚙️]
```
`🟢/🟡/🔴` — Финансовый пульс: индикатор здоровья бизнеса, тап → объяснение что влияет. Пульс — фича Модуля Розница: на тарифе Личный/ИП его в шапке нет.  
`[🔔]` — центр уведомлений: все события которые не попали в push (бейдж с числом непрочитанных).  
`[💬 ИИ]` — открывает чат с ИИ-ассистентом в нижнем листе.  
Пульс отображается **на всех экранах** (Касса, Поставщики, Персонал, Финансы) — постоянно виден без переходов.

**Главная зависит от роли:**
- **Бухгалтер** — полная версия (описана ниже)
- **Администратор** — без сумм кассы и баланса: чеклист его задач (заказы, записи на выплату), ожидаемые заказы, быстрые действия [📦 Заказ] [✍ Запись на выплату]
- **Сотрудник зала** — только заказы со статусами и последние поставки; никаких денег
- **Владелец** — режим просмотра: выручка, касса, долг, пульс, накопления — без кнопок ввода

---

**Чеклист дня** (под шапкой, всегда виден):
```
Сегодня:  ☑ Z-отчёт  ☐ 3 выплаты  ☑ Табель  ☐ Отчёт
```
Каждый пункт — тап → переход к нужному действию. Исчезает когда всё сделано.

---

**Баланс:**
```
Наличных в кассе
₽ 48 500                    Ср, 11 июн
Выручка сегодня      Свободно
₽ 62 000             ₽ 15 200
```
> «Свободно» = наличка − запланированные выплаты − сумма к пополнению накоплений

---

**Быстрые действия** (4 кнопки, самые частые операции):
```
[🧾 Z-отчёт]  [💸 Выплата]  [📦 Заказ]  [📤 Отчёт]
```
Открывают нужную форму прямо здесь — без перехода в другой раздел.

---

**Карточка Z-отчёта** (только если смена не заполнена):
```
┌──────────────────────────────────┐
│  🧾 Смена не закрыта             │
│  Внести Z-отчёт             [→] │
└──────────────────────────────────┘
```
Исчезает после заполнения. Исправить — в разделе «Касса», каждое изменение фиксируется в аудите.

---

**Карточка «Выплаты сегодня»** (только если есть, свёрнута):
```
┌──────────────────────────────────┐
│  Выплаты сегодня                 │
│  4 поставщика  •  ₽ 94 000  [↓] │
└──────────────────────────────────┘
```
Тап [↓] → нижний лист → тап на поставщике → действия:
```
Иванов П.  •  Долг ₽35К
[💸 Оплатить] [📅 Перенести] [✗ Не пришёл] [🗑 Отменить]
```

---

**Карточка «Заказы сегодня»** (только если есть):
```
┌──────────────────────────────────┐
│  Ожидаемые заказы                │
│  Молочка • Иванов П.   ~₽ 15К > │
│  Бакалея • ООО Альфа   ~₽ 22К > │
└──────────────────────────────────┘
```

---

**Карточка «Накопления»:**
```
┌──────────────────────────────────┐
│  Накопления                      │
│  Аренда    ████████░░  ₽42К/₽60К │
│  Налоги    █████░░░░░  ₽18К/₽45К │
│  Зарплата  ██████████  ₽90К/₽90К │
└──────────────────────────────────┘
```

---

**Состояние конца дня** (когда все задачи выполнены — вместо пустых карточек):
```
┌──────────────────────────────────┐
│  ✓ День закрыт                   │
│  Выручка ₽93 500  •  Свободно ₽15 200 │
│  [📤 Отправить отчёт владельцу]  │
└──────────────────────────────────┘
```

---

**Значки на навигации** (красные точки — не нужно помнить, видно сразу):
- 🔴 на иконке «Касса» → смена не закрыта
- 🔴 на иконке «Поставщики» → есть просроченная выплата
- 🔴 на иконке «Персонал» → табель не подтверждён

---

**Умные карточки** — карточка не показывается если нечего показывать.  
Нет выплат сегодня → карточки выплат нет. Нет заказов → карточки заказов нет.  
Главная чистая когда нечего делать — не давит пустотой.

**Пустое состояние** (первый запуск): только баланс + быстрые действия + приглашение начать.

**Принцип:** все действия раскрываются в нижние листы — основной экран не меняется.

**Крайние случаи Главной:**
- Баланс отрицательный → сумма красным со знаком «−», пульс автоматически 🔴
- Название магазина длиннее 20 символов → обрезается с «...»
- 10+ выплат сегодня → показываются первые 3, кнопка «и ещё 7...» раскрывает остальные
- Нет интернета при открытии → показываются кешированные данные + полоска «Офлайн · данные от ЧЧ:ММ»
- Бэкенд недоступен (обслуживание) → «🔧 Технические работы. Данные в безопасности. Скоро вернёмся»

## Экран 1а: Сводка по всем магазинам

**Зачем:** владелец и главный бухгалтер при входе видят всю сеть сразу — без выбора магазина.  
**Кто видит:** только аккаунты с доступом к нескольким магазинам. Обычный сотрудник этот экран не видит — попадает сразу в свой магазин.

**Состав:**

```
Все магазины              Ср, 11 июн   [+ Магазин]

Общая выручка сегодня     Общий долг поставщикам
₽ 186 000                 ₽ 420 000

─── Магазины ───
┌──────────────────────────────────┐
│  Way Market №1                 > │
│  Выручка ₽112К  •  Касса ₽48К   │
│  ⚠️ Смена не закрыта            │
└──────────────────────────────────┘
┌──────────────────────────────────┐
│  Way Market №2                 > │
│  Выручка ₽74К   •  Касса ₽31К   │
│  ✓ Смена закрыта                │
└──────────────────────────────────┘
```

Тап на магазине → переход внутрь, все экраны этого магазина (Главная, Касса, Поставщики...).  
Вернуться к сводке — кнопка «← Все магазины» в шапке.

**Карточка магазина показывает:**
- Название
- Выручка сегодня + наличка в кассе
- Статус смены (закрыта / не закрыта — чтобы не пропустить)

## Экран 2: Касса

**Зачем:** ввод Z-отчёта по смене и история всех смен. Сценарий 1.  
**Кто:** только Бухгалтер.

**Вкладки внутри раздела:** `[Новая смена]` `[История]`

---

### Вкладка «Новая смена»

```
Дата      [11.06.2025]
Смена     [Дневная ▼]       Кассир  [Иванова А. ▼]

─── По Z-отчёту ───
Наличные          ₽ [________]
Карта             ₽ [________]
Переводы (СБП)    ₽ [________]

─── Фактически в кассе ───
Наличные          ₽ [________]

─── Выплаты с кассы ───
[+ Добавить выплату]
  Инкассация  ₽ 20 000  [✕]
  Иман        ₽  3 500  [✕]

─── Итог ───
Расхождение: 🔴 −₽ 300
((Факт + Выплаты) − Z-отчёт)
Иман — тип выплаты, входит в «Выплаты» — не считается дважды.
Недостача → минус 🔴 · Излишек → плюс 🟡 · Ноль → ✓

[Сохранить смену]
```

После сохранения: карточка Z-отчёта на Главной исчезает.

---

### Вкладка «История»

Последние 7 дней — видно сразу. Дальше — свёрнуто по месяцам.

```
─── Сегодня, 11 июн ───
Дневная  Иванова А.  ₽62К  ✓ 0₽     >
Ночная   Петров В.   ₽31К  🔴 -₽300  >

─── Вчера, 10 июн ───
Дневная  Иванова А.  ₽58К  ✓ 0₽     >
...

─── Май 2025 ──────────────────── [↓] ───
─── Апрель 2025 ───────────────── [↓] ───
```

Тап на смене → детали: все введённые суммы + история исправлений (кто, когда, что изменил).  
Кнопка «Редактировать» — открывает форму с заполненными данными. Изменение фиксируется в аудите.

**Пустое состояние:** «Смены ещё не вводились. Начни с кнопки Z-отчёта на главной.»

**Крайние случаи Кассы:**
- Попытка создать Z-отчёт за дату+смену которая уже есть → открывается существующий с кнопкой «Редактировать» (не дублируется)
- Выплаты с кассы превышают наличные → жёлтое предупреждение, сохранение не блокируется (могут быть законные случаи)
- Дата в будущем → заблокировано, «Нельзя создать Z-отчёт за будущую дату»
- Кассир удалён из системы, но стоит в истории → отображается «Иванова А. (уволена)», история не стирается
- Исправил старый Z-отчёт → тост: «Балансы пересчитаны. Проверь свободный остаток на Главной»

### Вкладка «Зарплата»

Открывается в конце месяца. Показывает расчёт по каждому сотруднику.

```

             Аванс              -₽10 000
             Итого:              ₽23 000  [Выплатить]

             Аванс              -₽5 000
             Итого:              ₽25 240  [Выплатить]

```

**Пустое состояние:** «Добавь первого сотрудника чтобы начать вести табель.»

**Крайние случаи Персонала:**
- Авансы больше заработанной зарплаты → показываем «₽0 к выплате (аванс превысил заработок на ₽X — перенесено на следующий месяц)»; отрицательных выплат нет

- Двое сотрудников с одинаковым именем → система добавляет уточнение: «Иванова А. (1)» и «Иванова А. (2)»; предупреждение при создании
- Сотрудник весь месяц на больничном → зарплата ₽0, в строке: «Весь месяц больничный»

## Экран 9: ИИ-ассистент

**Зачем:** бухгалтер или владелец задаёт вопрос голосом или текстом — получает ответ на основе данных своего магазина.  
**Как открывается:** кнопка `[💬 ИИ]` в шапке Главной. Нижний лист поверх экрана.  
**Пакет ИИ.**

```
┌──────────────────────────────────────┐
│  💬 ИИ-ассистент              [✕]   │
│                                      │
│  ┌────────────────────────────────┐  │
│  │ Почему выручка упала на прошлой│  │
│  │ неделе?                        │  │
│  └────────────────────────────────┘  │
│                                      │
│  📊 Вижу снижение на 18% (пн–ср).    │
│  Возможные причины:                  │
│  • Понедельник: смена не закрыта     │
│    до 23:00 (Петров В.)             │
│  • Вторник: не было прихода молочки  │
│    от Иванова П.                     │
│  • Среда: расхождение -₽800         │
│                                      │
│  Хочешь посмотреть детали            │
│  по каждому дню?  [Да] [Нет]         │
│                                      │
│  ─────────────────────────────────   │
│  [🎤]  Спроси что-нибудь...   [→]   │
└──────────────────────────────────────┘
```

**Что умеет ИИ:**
- Отвечает на вопросы о данных магазина («Сколько мы заплатили Иванову за май?»)
- Объясняет аномалии («Почему расхождение именно в эти дни?»)
- Даёт прогнозы («Успеем накопить на аренду?»)
- Переходы: «Показать все смены Петрова» → открывает нужный экран

**Голосовой ввод:** микрофон → транскрипция → тот же ответ.

**Крайние случаи ИИ-ассистента:**
- ИИ-сервис недоступен → «ИИ временно недоступен. Ваши данные в порядке. Попробуйте позже» — не страшная ошибка
- Пользователь без Пакета ИИ тапнул [💬 ИИ] → показывается превью возможностей + «Подключить Пакет ИИ за ₽290/мес → [Попробовать 14 дней]»

## Примечание: планшет и компьютер

На широком экране (планшет, ПК) интерфейс перестраивается в два столбца:
- **Левая колонка** — навигация + список (поставщики, транзакции, сотрудники)
- **Правая колонка** — детали выбранного элемента

Бухгалтер на компьютере видит список поставщиков и карточку одновременно — без переходов между экранами. Все горячие клавиши (N, Z, P, /) работают только на широком экране.
