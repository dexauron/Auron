# 02 — Ключевые сценарии

> Статус: 🔄 на финальном согласовании.

---

## Сценарий 1: Утро бухгалтера — сверка кассы
**Кто:** Бухгалтер | **Сейчас:** ~1 час | **Цель:** 15 минут

1. Берёт Z-отчёт с кассы — дневная и ночная смена, имя кассира каждой смены
2. Вводит по каждой смене: наличка по Z-отчёту, карта (терминал), переводы
3. Вводит: фактическая наличка + все выплаты с кассы + Иман
4. Расхождение считается автоматически: `Z-отчёт − (Факт + Выплаты + Иман)`
5. Расхождение привязывается к кассиру смены
6. Видит список запланированных выплат на сегодня → резервирует
7. Видит свободный остаток

**Иман:** хозяйка берёт товар пробив на кассе без оплаты — записывается отдельно  
**Карта и переводы:** обычно не сверяются (терминал не ошибается), но опция сверки есть

---

## Сценарий 2: Выплата торговому
**Кто:** Бухгалтер | **В день:** 15–25 поставщиков

**Торговый пришёл:**
1. Ищет в справочнике по названию компании или имени → или создаёт нового прямо здесь
2. Видит текущий долг
3. Вводит: оплата наличкой / погашение долга / новый долг (сумма накладной)
4. Долг пересчитывается: `Новый долг = Старый + Взяли в долг − Погасили`
5. Доступная сумма уменьшается автоматически

**Торговый не пришёл в запланированный день:**
Приложение предлагает варианты:
- Перенести на завтра (та же сумма)
- Перенести на завтра и увеличить сумму
- Перенести на конкретную дату
- Освободить — вернуть в свободные деньги

**Возврат поставщику:** отдельная строка — уменьшает долг или возвращает наличку

---

## Сценарий 3: Заказ (администратор)
**Кто:** Администратор | **Источник:** лично с торговым или из группового чата сотрудников

**Отделы:** настраиваются под свой магазин (Молочка, Заморозка, Бакалея, Химия и т.д.)

**Создание заказа:**
1. Поставщик, отдел, сумма заказа, ожидаемая дата прихода → Статус: Ожидается

**Товар пришёл:**
1. Находит заказ, отмечает: дата прихода + фактическая сумма (часто ≠ заказу — норма)
2. Если есть свободные деньги → платит сразу (наличка)
3. Если нет → в долг, бухгалтер планирует выплату

**Видно по каждому заказу:** дата заказа / поставщик / отдел / сумма заказа / ожидаемая дата / статус / фактическая сумма

---

## Сценарий 4: Ежемесячные накопления
**Кто:** Бухгалтер

**В начале месяца:** вводит плановые суммы по статьям:
- Аренда (фиксированная), Зарплата (примерно), Налоги (примерно), Коммуналка, прочее
- Приложение считает сколько откладывать в день до конца месяца

**Каждый день:** план / отложено / осталось / дней до конца месяца — по каждой статье

**Если факт > план:** доплачивают разницу из свободных денег  
**Логика:** лучше накопить 100к и доложить 20к, чем искать 120к за раз

---

## Сценарий 5: Сотрудники, табель и зарплата
**Кто:** Бухгалтер / Администратор

**Управление сотрудниками:**
- Добавить: имя, должность, ставка (за день или за час)
- Уволить: статус «уволен» с датой, история сохраняется, из активного списка уходит
- Заменить уволенного новым в любое время

**Табель рабочих дней (каждый день):**
- По каждому сотруднику отмечается: вышел / выходной / больничный / прогул / отпуск
- Фиксируется количество часов если почасовая оплата
- В конце месяца видно: сколько дней/часов отработал каждый

**Расчёт зарплаты (автоматически):**
- `Зарплата к выплате = Ставка × Отработанные дни/часы − Авансы`
- Бухгалтер видит итоговую сумму по каждому сотруднику перед выплатой
- Может скорректировать вручную если нужно

**Авансы:**
- Кому, сколько, дата — автоматически вычитается из итоговой зарплаты

**Выплата:** кому, сколько, дата → история по каждому сотруднику

## Сценарий 11: Цели и планирование
**Кто:** Владелец (ставит цели) + Бухгалтер (видит прогресс)

**Создание цели:**
- Тип: выручка / расходы / прибыль / долг поставщикам / накопления
- Направление: достичь суммы X / снизить до X / не превысить X
- Период: месяц / квартал / год / произвольный диапазон
- Примеры: «выручка 8 млн за месяц», «расходы не больше 1.5 млн», «погасить долг до 3 млн»

**Прогресс (обновляется автоматически каждый день):**
- Цель / Факт на сегодня / Осталось / Тренд (успеваем или нет)
- Если не успеваем — сколько нужно в день чтобы догнать

**В конце периода:**
- Итог: план vs факт, разница в числах и процентах
- История закрытых целей — видно динамику по месяцам

---

## Сценарий 6: Прочие расходы и списания
**Кто:** Бухгалтер

- **Прочие расходы** (ГСМ, расходные материалы и т.д.): сумма + комментарий одной строкой
- **Списания продукта:** сумма + что списано (сам товар остаётся в 1С, нам нужна только финансовая сторона)

## Сценарий 7а: Дневной отчёт
**Кто:** Бухгалтер (отправляет) + Владелец (получает) | **Когда:** в конце каждого рабочего дня

Автоматически формируется из данных дня — бухгалтер ничего не набирает вручную.

**Содержание отчёта:**

💰 **Выручка**
- Итого: наличка / карта / переводы
- По сменам: кассир, сумма, расхождение

💸 **Движение денег**
- Выплачено поставщикам
- Свободный остаток в кассе
- Общий долг поставщикам: было → стало

📦 **Товары и заказы**
- Какие заказы пришли сегодня и на какую сумму
- Новые долги за принятые товары

👥 **Персонал**
- Кто работал / кто отсутствовал
- Авансы выданные за день

📊 **Накопления и цели**
- Прогресс по накоплениям (аренда, налоги, зарплата)
- Прогресс по активным целям: «Цель ₽8 млн — выполнено 67%»
- Прогноз: «При текущем темпе — закроешь месяц на ₽7.4 млн»

📈 **Сравнение**
- Выручка сегодня vs тот же день прошлой недели
- Выручка сегодня vs средний день текущего месяца

🔔 **Флаги** (показываются только если есть проблема)
- ⚠️ Расхождение по кассе
- ⚠️ Поставщик не пришёл — выплата не проведена
- ⚠️ До конца месяца X дней, на [статья] не хватает ₽Y

💬 **Итоговая строка** — одна фраза-вердикт дня, генерирует ИИ:
> «Хороший день: выручка выше среднего, расхождений нет, все выплаты проведены» *(платный тариф)*

**Форматы отправки:**
- Текст → WhatsApp / Telegram (через системный шейр, одним тапом)
- PDF → скачать или отправить на почту
- Живая ссылка (платный тариф) — обновляется, владелец открывает в браузере

**Кнопка** «Отправить итог дня» появляется на Главной после заполнения Z-отчёта.

## Сценарий 7: Конец месяца — итоговый отчёт
**Кто:** Бухгалтер (формирует) + Владелец (читает)

**Отчёт включает:**
- Все доходы (наличка + карта + переводы)
- Все расходы по категориям
- Списания
- Валовая прибыль
- Средние показатели (выручка в день, расход в день)
- Долг поставщикам на конец месяца
- Остаток наличных
- **Рентабельность** — `(Выручка − Себестоимость) / Выручка × 100%`. Каждый магазин настраивает свой целевой процент. Себестоимость берётся из импорта 1С (см. ниже).

**Кассовая книга:** хронологический журнал всех приходов и расходов за месяц. Формируется автоматически из всех записей, можно распечатать. Страховка при вопросах от налоговой.

**Экспорт:** весь отчёт и любой раздел выгружается в Excel

## Сценарий 12: Учёт товаров и рентабельность
**Кто:** Бухгалтер / Администратор | **Настраивается при регистрации магазина**

Режим выбирается один раз при настройке: **1С** или **Ручной**. Можно поменять позже.

---

### Режим А: Импорт из 1С (крупный магазин, 100+ позиций)
**Зачем:** Ручной ввод сотен товаров нереален — 1С уже хранит всё нужное.

1. В 1С формируется отчёт по продажам или остаткам → экспорт в Excel
2. В приложении: «Импорт из 1С» → вставить файл или скопировать содержимое
3. Приложение читает нужные столбцы (товар, себестоимость, количество, сумма продаж)
4. Рентабельность пересчитывается автоматически

---

### Режим Б: Ручной учёт (маленький магазин, до ~50 позиций)
**Для кого:** магазины без 1С — одежда, обувь, товары для дома и т.д.

**Справочник товаров:**
- Добавить товар: название, закупочная цена, цена продажи → наценка % считается автоматически
- Редактировать, архивировать (убрать из активных, история остаётся)

**Приход товара:**
- Поставщик, товар, количество, сумма → долг поставщику растёт (как в Сценарии 2)

**Продажи (опционально):**
- Зафиксировать продажу: товар, количество → выручка добавляется к дневному итогу
- Или просто вести выручку через Z-отчёт (как в Сценарии 1) — без разбивки по товарам

**Рентабельность:**
- `(Выручка − Себестоимость проданного) / Выручка × 100%`
- Видно по каждому товару и общая по магазину

**Что НЕ делает:** полноценный складской учёт — остатки только примерные, для точного учёта нужна 1С или другая система.

---

## Сценарий 8: Владелец смотрит
**Кто:** Владелец | **Устройство:** телефон | **Только читает**

- Выручка сегодня / неделя / месяц (наличка + карта + переводы)
- Расхождения — по смене и по кассиру
- Наличка в кассе сейчас
- Долг поставщикам (общий + по каждому)
- Накопления: отложено / осталось
- Заказы: ждём / пришло / на какую сумму
- Месячный отчёт когда закрыт

---

## Сценарий 9: Уведомления
**Для кого:** Бухгалтер и Администратор (на телефон)

**Тип 1 — Плановые** (рассылаются утром по расписанию):
- «Сегодня запланированы выплаты: [список с суммами]»
- «Ожидается приход заказа: [поставщик, отдел, сумма]»

**Тип 2 — Напоминания о незакрытых задачах** (если задача не выполнена до определённого времени):
- «Z-отчёт за дневную смену ещё не заполнен» (пример: в 15:00)
- «Табель за сегодня не подтверждён» (пример: в 21:00)
- «Иванов П. должен был прийти сегодня — выплата не проведена»
- «До конца месяца 3 дня, на аренду не хватает ₽12 000»

**Настройки уведомлений:** каждый тип можно включить/выключить, время напоминаний — настраивается под режим работы магазина.

## Сценарий 10: Журнал изменений (аудит)
**Видят:** Владелец и Бухгалтер

Любое изменение любой записи фиксируется навсегда:
- Кто изменил / когда / во сколько
- Что было до / что стало после

Исправить запись можно — след остаётся всегда.

---

## Доступ и роли

### Структура аккаунта
- Один аккаунт владельца содержит **несколько магазинов** (компаний)
- Каждый магазин — изолированная среда: сотрудники одного магазина не видят данные другого
- Владелец видит **все свои магазины** из одного аккаунта — переключается между ними
- Владелец может дать другому аккаунту **сквозной доступ** ко всем магазинам (управляющий, доверенный бухгалтер сети) — такой человек видит всё как владелец, но не управляет настройками

### Сотрудники и роли
- У каждого своя учётная запись (логин + пароль)
- Четыре роли: **Бухгалтер** / **Администратор** / **Сотрудник зала** / **Владелец**
- Роли можно комбинировать: один человек может быть Бухгалтером + Администратором одновременно — видит объединение всех своих прав
- **Только Владелец** создаёт аккаунты, назначает роли, меняет и отзывает доступ
- Владелец — особая роль, её нельзя передать или совместить с другими

**Что видит каждая роль:**

| Раздел | Бухгалтер | Администратор | Сотрудник зала | Владелец |
|--------|-----------|---------------|----------------|----------|
| Касса (Z-отчёт) | ✅ | ❌ | ❌ | 👁 |
| Поставщики (долги, выплаты) | ✅ | 📝 записать | ❌ | 👁 |
| Заказы | ✅ | ✅ | 👁 статус | 👁 |
| Товары (каталог, история) | ✅ | ✅ | 👁 без цен закупки | 👁 |
| Персонал и зарплаты | ✅ | ❌ | ❌ | 👁 |
| Накопления и цели | ✅ | ❌ | ❌ | 👁 |
| Финансовый отчёт | ✅ | ❌ | ❌ | 👁 |
| Управление доступом | ❌ | ❌ | ❌ | ✅ |

> 👁 = только чтение, ✅ = полный доступ, 📝 = ограниченные действия, ❌ = нет доступа

**Управление доступом (в настройках, только для Владельца):**
1. Добавить сотрудника: имя, телефон/email → система отправляет приглашение
2. Назначить роль(и): Бухгалтер / Администратор / Сотрудник зала (можно несколько)
3. Несколько человек могут иметь одинаковую роль (два бухгалтера, два администратора) — каждый со своим аккаунтом и своей историей действий
4. Изменить роли в любой момент
5. Отключить доступ (аккаунт сохраняется, войти нельзя)

**Почему важен аудит при нескольких сотрудниках:** когда двое работают в одной системе, журнал изменений показывает конкретно кто и что сделал — не «бухгалтер изменил», а «Марина изменила сумму с 15 000 на 18 000 в 14:32».

## Первый запуск (одноразово)
- Ввести список поставщиков с текущими суммами долга
- Ввести список сотрудников с зарплатами
- Настроить отделы
- Настроить статьи ежемесячных расходов

---

## Что НЕ входит
- Работа с 1С (остатки, ТСД, ценники)
- Управление расписанием сотрудников
- Складской учёт
- Интеграция с кассой напрямую
