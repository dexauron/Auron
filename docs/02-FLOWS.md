## Сценарий 1: Утро бухгалтера — сверка кассы
**Кто:** Бухгалтер  
**Когда:** Каждое утро, первое дело  
**Сколько занимает сейчас:** ~1 час в Excel  
**Цель в приложении:** до 15 минут

**Шаги:**
1. Берёт Z-отчёт (чек с кассы) — отдельно за дневную и ночную смены
2. Вводит в приложение: сумма Z-отчёта наличкой по каждой смене
3. Вводит: фактические деньги которые отдала касса + все выплаты сделанные с кассы во время смены
4. Приложение считает расхождение автоматически:
   > `Расхождение = Z-отчёт − (Факт + Выплаты с кассы + Иман)`
5. Видит список запланированных выплат на сегодня (записи торговых + ежемесячные накопления)
6. Резервирует эти суммы — они «уходят» из доступных денег
7. Видит остаток — свободные деньги которые можно тратить

**Что такое Иман:** хозяйка магазина берёт товар с кассы (пробит, но не оплачен) — записывается отдельной строкой чтобы касса показывала корректно

**Формула доступных денег:**
> `Доступно = Вся наличка − Записи на выплату − Накопления на постоянные расходы`

## Сценарий 2: Выплата торговому
**Кто:** Бухгалтер  
**Когда:** В течение дня, каждый приход торгового  
**Сколько торговых в день:** ~15–25

**Шаги (торговый по записи):**
1. Бухгалтер открывает список торговых, ищет по названию компании или имени
2. Если нет в списке — создаёт прямо тут (название компании + имя представителя)
3. Видит текущий долг этого поставщика
4. Вводит три числа по сегодняшнему визиту:
   - **Оплата наличкой** — за товар привезённый сегодня, платим сразу
   - **Погашение долга** — платим часть накопленного долга
   - **Новый долг** — товар взяли в долг сегодня (сумма накладной)
5. Долг пересчитывается: `Долг новый = Долг старый + Новый долг − Погашение долга`
6. Доступная сумма уменьшается на выплаченное
7. Торговый расписывается на накладной — физически, не в приложении

**Шаги (торговый пришёл внезапно):**
1. Проверяем доступный остаток
2. Если деньги есть — проводим выплату как обычно
3. Если денег нет — создаём запись на будущее: дата + сумма
4. В назначенный день деньги заранее зарезервированы

**Оплаченные накладные:** когда долг погашен — физически накладная переходит в папку «оплаченных». В приложении это отражается в истории платежей.

## Сценарий 3: Заказ у торгового (администратор)
**Кто:** Администратор (или сотрудники зала → администратор)  
**Когда:** Когда торговый пришёл в магазин или по телефону  

**Источники заказа:**
- Администратор делает заказ лично пока торговый в магазине
- Сотрудники зала делают заказ и пишут в групповой чат → администратор переносит в приложение

**Шаги (создание заказа):**
1. Выбирает поставщика из справочника (или создаёт нового)
2. Вводит: отдел, сумма заказа, ожидаемая дата доставки
3. Заказ появляется в списке «Ожидаем»

**Шаги (товар пришёл):**
1. Оператор принимает товар через ТСД в 1С — это не в приложении
2. Администратор находит заказ в списке
3. Отмечает: пришло / дата прихода / фактическая сумма накладной
4. Если сумма отличается от заказа — записывает фактическую (это норма)
5. Администратор сообщает бухгалтеру — нужно запланировать выплату

**Что видно по заказу:**
- Дата заказа, поставщик, отдел, сумма заказа
- Ожидаемая дата прихода
- Статус: Ожидается / Пришло / Оплачено
- Фактическая сумма когда пришло

## Сценарий 4: Ежемесячные накопления
**Кто:** Бухгалтер  
**Когда:** Один раз в начале месяца — планирование. Каждый день — откладывание.

**В начале месяца:**
1. Бухгалтер вводит плановые суммы расходов на месяц:
   - Аренда, Зарплата, Налоги, Коммуналка — и другие постоянные
2. Приложение считает сколько нужно откладывать в день (÷ на рабочие дни)

**Каждый день:**
1. После сверки кассы — часть денег резервируется под накопления
2. На экране видно по каждой статье: план / отложено / осталось накопить
3. Когда накопили нужную сумму — статья закрывается

**В конце месяца:**
- Бухгалтер выплачивает из накопленного: аренду, зарплаты, налоги
- Цикл начинается заново

## Сценарий 5: Владелец смотрит как идут дела
**Кто:** Владелец  
**Когда:** В любое время, с телефона  
**Действий не совершает — только читает**

**Что видит:**
- Выручка сегодня (наличка + карта + переводы) по сменам
- Расхождения кассы за день
- Наличка в кассе прямо сейчас
- Общий долг поставщикам + список с суммой по каждому
- Накопления: что отложено, сколько не хватает
- Заказы: что ждём, что пришло, на какую сумму
