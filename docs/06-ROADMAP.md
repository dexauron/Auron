# 06 — Дорожная карта

> Это план развития компании, не жёсткий график. Действуем по ситуации и обстоятельствам, но движемся по этим ступеням.

---

## Шаг 1 — Доказать спрос
**Цель: 50–100 платящих магазинов**

- Запустить Auron Finance для розницы (модуль Way Market)
- Довести до идеала — каждый сценарий работает без ошибок
- Way Market №2 — первый клиент, живой тест в реальных условиях
- Собрать обратную связь, устранить все боли
- Понять: платят ли незнакомые люди, не только знакомые

**Команда:** только основатель  
**Инструменты:** Auron Finance v1.0, Claude API для ИИ

---

## Шаг 2 — Собрать выручку и нанять команду
**Цель: 300–500 платящих магазинов**

- При 100 магазинах по ₽890 = **₽89 000/мес** — первая выручка
- Нанять первого разработчика
- Нанять поддержку клиентов
- Улучшать продукт быстрее с командой
- Начать думать о второй нише

**Когда переходить:** стабильная выручка покрывает зарплату первого сотрудника

---

## Шаг 3 — Вторая ниша
**Цель: доказать что модель масштабируется**

- Выбрать следующую нишу по результатам обратной связи (кафе / салоны / ИП — по ситуации)
- Запустить второй модуль на той же платформе
- Доказать что архитектура работает: новый модуль добавился без переписывания старого

**Команда:** 2–3 человека  
**Ключевое решение:** какую нишу брать — решать по данным, не заранее

---

## Шаг 4 — Платформа
**Цель: Auron Finance → многоотраслевая платформа**

- Несколько ниш, несколько модулей работают параллельно
- Сторонние разработчики начинают делать свои модули
- Auron Marketplace — магазин модулей для разных сфер
- Международный рынок если есть спрос

**Команда:** полноценная компания  
**Когда переходить:** когда платформенная архитектура доказала себя на 3+ нишах

---

## Принцип

Каждый шаг финансирует следующий. Не тратим инвестиции — зарабатываем на текущем шаге и вкладываем в следующий. Масштаб заложен в архитектуре с первого дня — но не торопим его раньше времени.
